ATS Style Guide

Use this style guide to build any line of business application.

Current Version: 2.0.0

Get Started

  Getting Started

Make use of this basic HTML template. It should have everything you need to get started.

Note: you must host the respond-proxy.js and respond-proxy.gif files locally in your project if you need IE8 support. If IE8 support is not required you can ignore the conditional comment in this example. 

<!DOCTYPE html>
<html>
    <head> 
       <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>A Page</title>
        <link rel="stylesheet" href="//az452370.vo.msecnd.net/styleguide/css/2.0.0/atsstyles.min.css"/>

        <!--[if lt IE 9]> 
           <link rel="stylesheet" href="//az452370.vo.msecnd.net/styleguide/css/2.0.0/ie.min.css"/> 
           <script type="text/javascript" src="//az452370.vo.msecnd.net/styleguide/js/2.0.0/ie.min.js"></script> 
           <link href="//az452370.vo.msecnd.net/styleguide/respond-proxy.html" id="respond-proxy" rel="respond-proxy" /> 
           <link href="Path to your local respond-proxy.gif" id="respond-redirect" rel="respond-redirect" />  
          <script type="text/javascript" src="Path to your local respond-proxy.js"></script> 
       <![endif]-->
       </head>
        <body> 
           <div class="container"> 
               ... 
           </div>
           <script type="text/javascript" src="//az452370.vo.msecnd.net/styleguide/js/2.0.0/ats.min.js"></script>
       </body>
</html>

  Azure CDN Usage

All of the files for the ATS style guide are stored on our Azure CDN. It uses a CDN on top of blob storage to serve up our content. To take advantage of CDN caching it's recommended that you use the CDN urls, but you can also use the blob storage urls directly.
